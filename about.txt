              СПРАВКА: РАБОТА МОДУЛЕЙ "ПОД КАПОТОМ" (ABOUT - WINDOWS)
========================================================================

В данном документе приведено техническое описание внутренней логики работы 
каждого модуля программного комплекса Windows Security Checker.

------------------------------------------------------------------------
1. Главный хаб управления (hub.ps1)
------------------------------------------------------------------------
* Проверка привилегий: Использует класс `[Security.Principal.WindowsPrincipal]` 
  и удостоверение `WindowsIdentity::GetCurrent()`, сравнивая текущую роль с 
  `Administrator`. Прерывает выполнение в случае отсутствия прав администратора.
* Инициализация структуры: Автоматически проверяет наличие и создает при 
  отсутствии рабочие папки `report/`, `tools/`, `service/`, `scripts/`.
* Контроль настроек СЗИ: Считывает и сохраняет преференции целевой системы 
  защиты информации (Dallas Lock, Secret Net Studio или Без СЗИ) в текстовый 
  файл `service/szi_settings.conf`.
* Диспетчеризация: Проверяет наличие соответствующих PowerShell-скриптов 
  в каталоге `scripts/` и выполняет их запуск в текущей сессии.

------------------------------------------------------------------------
2. Аудит безопасности и комплаенса (scripts/security_checker.ps1)
------------------------------------------------------------------------
* Контроль соответствия классу защищенности:
  - Считывает эталонные требования (длина/история/срок действия паролей, 
    порог блокировки, таймауты, очистка Pagefile, защита LSA, логи аудита) 
    из конфигурационного файла `service/compliance_standards.conf`.
  - При отсутствии файла переходит на встроенные жестко заданные лимиты.
* Сбор локальной политики учетных записей:
  - Вызывает встроенную системную утилиту `secedit.exe` с аргументами:
    `/export /cfg [temp_file] /areas SECURITYPOLICY`
    для извлечения параметров политики безопасности в файл конфигурации INI.
  - Считывает параметры `MinimumPasswordLength`, `PasswordHistorySize`, 
    `MaximumPasswordAge`, `LockoutBadCount`, `LockoutDuration`.
  - Для обеспечения независимости от языковой локали (русская/английская ОС) 
    использует резервный опрос локальной SAM через ADSI (`[ADSI]"WinNT://$env:COMPUTERNAME"`).
* Опрос параметров сессии и ядра в системном реестре:
  - Таймаут неактивности сессии: проверяет ключ `InactivityTimeoutSecs` в 
    `HKLM:\Software\Microsoft\Windows\CurrentVersion\Policies\System`. 
    При его отсутствии опрашивает параметры заставки (`ScreenSaveActive`, 
    `ScreenSaveTimeOut`) в профиле текущего пользователя (`HKCU:\Control Panel\Desktop`).
  - Очистка Swap/Pagefile: проверяет Dword-параметр `ClearPageFileAtShutdown` 
    в ветке `HKLM:\SYSTEM\CurrentControlSet\Control\Session Manager\Memory Management`.
  - Защита LSA: проверяет параметр `RunAsPPL` в `HKLM:\SYSTEM\CurrentControlSet\Control\Lsa`.
  - Блокировка консоли: проверяет параметр `ScreenSaverIsSecure` в HKCU.
* Сбор лога подключенных USB-устройств:
  - Опрашивает ветку реестра `HKLM:\SYSTEM\CurrentControlSet\Enum\USBSTOR` 
    (содержит сведения обо всех подключенных когда-либо USB-накопителях).
  - Извлекает имя устройства, серийный номер, описание драйвера и дружественное 
    имя (`FriendlyName`).
* Формирование перечня ПО:
  - Читает подразделы `Uninstall` в ветках:
    * `HKLM:\Software\Microsoft\Windows\CurrentVersion\Uninstall` (64-бит ПО)
    * `HKLM:\Software\Wow6432Node\Microsoft\Windows\CurrentVersion\Uninstall` (32-бит ПО)
    * `HKCU:\Software\Microsoft\Windows\CurrentVersion\Uninstall` (ПО текущего пользователя)
  - Фильтрует системные компоненты и обновления, формируя перечень ПО с именем, 
    версией, издателем и датой установки. Сохраняет в `report/installed_software.txt`.
* Генерация отчетов:
  - Записывает детальный текстовый отчет сравнения параметров с рекомендациями в 
    `report/compliance_report_[КЛАСС]_[ДАТА_ВРЕМЯ].txt`.
  - Передает собранные данные в `service/report_generator.ps1` для компиляции HTML-отчета.

------------------------------------------------------------------------
3. Запуск внешних утилит (scripts/scanoval_checker.ps1)
------------------------------------------------------------------------
* Поиск утилит: Выполняет поиск исполняемых файлов WinAudit.exe, usbdeview.exe, 
  HWInfo64.exe (или HWInfo32.exe) и ScanOVAL.exe в папке `tools/` и ее подпапках.
* Сбор данных WinAudit: Запускает утилиту в командном режиме:
  `WinAudit.exe /r=gsoPxuTUeERNtnzDaIbMpmidcSArCOHG /o=HTMLi /f=report\winaudit_report.html`
  для создания интерактивного отчета аудита конфигурации Windows.
* Сбор данных USBDeview: Запускает утилиту для генерации HTML-отчета:
  `usbdeview.exe /shtml report\usbdeview_report.html`
* Сбор данных HWInfo: Запускает утилиту с ключом `/log` для формирования отчета 
  по оборудованию.
* Сканер ScanOval: Находит XML-файл базы уязвимостей ФСТЭК в папке `service/`. 
  Запускает `ScanOVAL.exe` в режиме CLI с ключами `-o` (база), `-r` (результаты XML), 
  `-h` (отчет HTML). При отсутствии базы запускает графический интерфейс ScanOval.

------------------------------------------------------------------------
4. Выбор СЗИ и проверка антивирусов (scripts/antivirus_installer.ps1)
------------------------------------------------------------------------
* Оценка статуса антивирусной защиты:
  - Windows Defender: Опрашивает состояние службы `WinDefend`.
  - Kaspersky/Dr.Web: Сканирует список служб ОС на наличие процессов/служб, 
    содержащих в названии или описании "Kaspersky", "avp", "drweb". Выводит 
    статус (запущена/остановлена) для каждой обнаруженной службы.
* Контроль состояния СЗИ от НСД:
  - Dallas Lock: Проверяет наличие раздела реестра `Dallas Lock` и активность 
    служб защиты (службы, начинающиеся с `Dl` или содержащие `Dallas Lock` в описании).
  - Secret Net Studio (SNS): Проверяет наличие разделов реестра `Security Code` 
    и активность служб, содержащих в имени `sns` или `Secret Net`.
* Поиск установочных пакетов: Сканирует все съемные тома на наличие файлов 
  установки по маскам `kes*.exe`, `kes*.msi`, `drweb*.exe`, `dl*.exe`, `sns*.exe`.

------------------------------------------------------------------------
5. Менеджер зависимостей (scripts/repo_manager.ps1)
------------------------------------------------------------------------
* Проверка сети: Выполняет попытку отправки ICMP-эхо запроса к `8.8.8.8` и 
  HTTP-запроса HEAD к серверу БДУ ФСТЭК для подтверждения наличия интернета.
* Автозагрузка: Скачивает zip-архивы утилит с официальных серверов (NirSoft, 
  HWInfo, GitHub), распаковывает их с помощью штатного командлета 
  `Expand-Archive` и раскладывает исполняемые файлы в соответствующие подпапки `tools/`.
* Загрузка баз ФСТЭК: Выполняет попытку скачать актуальную OVAL-базу уязвимостей 
  с сайта ФСТЭК (`https://bdu.fstec.ru/files/documents/oval.zip`), сохраняя и 
  распаковывая ее в каталог `service/`.
* Автономный поиск: Сканирует съемные накопители на наличие бинарных файлов 
  утилит для их автоматического копирования в папку `tools/`.
